从用户输入url地址到页面渲染数据中间发生了什么？

我们都知道互联网是在 TCP/IP 协议族的基础上运作的，http 协议只是其中的一个子集。因此，首先将从 TCP/IP 协议族展开。

TCP/IP 协议族

分层管理

按照层次分为：应用层、传输层、网络层、链路层。好处是分而自治。

通信传输流

TCP/IP 协议族进行网络通信时，会通过分层顺序与 Web 服务器进行通信。

发送端从应用层往下走，接收端则从应用层往上走。

首先作为发送端的客户端在应用层，通过 http 协议发送想看某一 url 地址的 http 请求。
接着，传输层 TCP 协议把从应用层处收到的 http 请求报文进行分割，并在各个豹纹上打上标记序号及端口号发给网络层。
在网络层，IP 协议增加作为通信目的地的 MAC 地址后转发给链路层。

在链路层接收到完整的请求数据为：以太网首部+IP 首部+TCP 首部+http 数据。

发送端传输数据时，每经过一层会被打上一个属于该层的首部信息。
这种把信息包装起来的方法称为封装。

接收端接收数据时，每经过一层会把对应的首部去掉。
接收端的 Web 服务器在链路层接收到数据，按顺序网上层发送。当传输到应用层，才算真正接收到客户端发送过来的请求。

各种协议在通信传输流中的作用

DNS 服务（负责域名解析）

DNS 服务是和 http 协议一样处于应用层的协议，提供域名到 IP 地址的解析服务。
DNS 协议提供通过域名查找 IP 地址，或者逆向的从 IP 地址反查域名的服务。

TCP 协议（确保可靠性）

TCP 协议为了可靠的传送大数据，将数据进行了分割，并且通过三次握手机制确认数据最终是否送达到接收端。

三次握手

握手过程中使用了 TCP 的标志——SYN 和 ACK。
发送端首先发送一个带有 SYN 标志的数据包给对方。
接收端收到后，回送一个带有 SYN/ACK 标志的数据包，以表示传达确认信息。
发送端再回送一个带有 ACK 标志的数据包，代表握手结束。

若在握手过程某个阶段莫名中断了，TCP 协议会再次以相同的顺序发送相同的数据包。

IP 协议（负责传输）

IP 协议的作用是把数据包传送给对方。
必须满足 IP 地址和 MAC 地址正确。

IP 地址指明了节点被分配到的地址，MAC 地址是网卡所属的固定地址。
IP 地址和 MAC 地址进行配对。
IP 地址可以变换，但 MAC 地址基本上不会更改。

通信双方不在一个局域网时，需要经过多台计算机和网络设备中转才能连接到对方。
中转时，只能利用下一个中转设备的 MAC 地址来搜索下一个中转目标。
根据 IP 地址反查出 MAC 地址的协议是 ARP 协议。
